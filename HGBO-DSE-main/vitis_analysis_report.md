# Vitis HLS 2025.1 与 HGBO-DSE 项目优化效果比较分析报告

## 1. 分析概述

本报告分析了Vitis HLS 2025.1的合成结果与HGBO-DSE项目的优化效果比较方法。

项目使用的资源权重配置：

- **WFF (FF权重)**: 0.25

## 4. 归一化PPA指标计算

### 4.1 基准值设置
使用与原始aes_aes_params.yaml相同的基准值：
- 基准功耗: 0.361 W
- 基准延迟: 713 cycles
- 基准时钟: 7.742 ns
- 基准LUT: 8,793
- 基准FF: 10,024
- 基准DSP: 0
- 基准BRAM: 4

### 4.2 归一化结果
由于Vitis HLS 2025.1的结果与基准值完全相同，所有归一化指标都为1.0：
- 归一化功耗: 1.0000
- 归一化延迟: 1.0000
- 归一化关键路径: 1.0000
- 归一化面积: 1.0000

## 5. 比较方法说明

### 5.1 归一化指标含义
- **< 1.0**: 表示比基准性能更好（更优）
- **= 1.0**: 表示与基准性能相同
- **> 1.0**: 表示比基准性能更差

### 5.2 多目标优化比较
对于多目标优化（Power, Latency, Critical Path, Area），应该：
1. 使用Pareto前沿分析
2. 比较不同算法找到的Pareto最优解集
3. 分析解集的覆盖性和分布性

## 6. 与HGBO-DSE项目比较的建议

### 6.1 运行HGBO-DSE优化
```bash
# 使用相同的基准配置运行HGBO-DSE
python bome/hls_dse.py --benchmark aes --algorithm sa --config config/MachSuite/aes_aes_vitis2025_params.yaml
```

### 6.2 比较方法
1. **单目标比较**: 比较归一化后的综合评分
2. **多目标比较**: 使用Pareto前沿分析
3. **收敛性比较**: 比较优化算法的收敛速度和稳定性

### 6.3 评估指标
- **超体积指标 (Hypervolume)**: 衡量Pareto前沿的质量
- **IGD指标 (Inverted Generational Distance)**: 衡量解集与真实Pareto前沿的距离
- **覆盖率**: 衡量一个算法找到的解集覆盖另一个算法解集的程度

## 7. 结论

1. **Vitis HLS 2025.1结果**: 与基准配置完全相同，归一化指标为1.0
2. **比较基准**: 可以作为HGBO-DSE项目优化的基准线
3. **优化目标**: HGBO-DSE项目应该找到归一化指标 < 1.0 的解决方案
4. **评估标准**: 数值越小表示性能越好，这是正确的比较方向
